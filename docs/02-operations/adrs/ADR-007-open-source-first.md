## ADR-007: Free & Open-Source First Strategy

## Status

Accepted

## Context

Code and Core aims to remain accessible, cost-effective, transparent, and community-driven.

Choosing open technologies reduces vendor lock-in and lowers operating costs.

## Decision

When practical, Code and Core will prefer **free and open-source software** over proprietary alternatives.

Exceptions may be made when proprietary software provides significant technical, legal, security, or operational advantages.

## Rationale

This approach:

- Lowers costs.
- Encourages transparency.
- Supports learning.
- Promotes community contribution.
- Reduces vendor dependency.

## Consequences

### Positive

- Lower operating expenses.
- Greater flexibility.
- Easier experimentation.

### Negative

- Some commercial tools may still be required for specialized needs.

## Alternatives Considered

- Commercial-first ecosystem.
- Mixed ecosystem without clear guidance.

Rejected because they increase long-term dependency and cost.
