## ADR-006: Professional, Clear & Educational Writing

## Status

Accepted

## Context

Documentation represents Code and Core's knowledge and professionalism. Inconsistent writing styles reduce clarity and create confusion.

## Decision

All official documentation shall follow a **Professional, Clear, and Educational** writing style.

Documentation should:

- Explain concepts before implementation.
- Use plain language whenever practical.
- Prefer active voice.
- Define technical terms on first use.
- Maintain a respectful and objective tone.

## Rationale

A consistent writing style:

- Improves readability.
- Supports learning.
- Reduces ambiguity.
- Creates a consistent organizational voice.

## Consequences

### Positive

- Consistent documentation.
- Better onboarding.
- Improved learning experience.

### Negative

- Requires authors to follow writing standards.

## Alternatives Considered

- Individual writing styles.
- Highly academic writing.
- Informal conversational writing.

Rejected because they reduce consistency.
