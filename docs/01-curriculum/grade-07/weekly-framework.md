## Grade 7 Weekly Learning Framework

## Purpose

The Grade 7 Weekly Learning Framework establishes a consistent instructional model for every lesson throughout the academic year. It supports active learning, problem solving, collaboration, and engineering practices while maintaining predictable classroom routines.

---

## Weekly Learning Cycle

Every instructional week follows the same learning process.

| Day | Focus | Objective |
| ----- | ------- | ----------- |
| Day 1 | Learn | Introduce new concepts and demonstrations |
| Day 2 | Practice | Guided coding and classroom activities |
| Day 3 | Build | Hands-on projects and laboratory work |
| Day 4 | Improve | Testing, debugging, and peer collaboration |
| Day 5 | Present & Reflect | Assessment, documentation, and reflection |

---

## Day 1 — Learn

## Objectives

Students will:

- Learn new concepts.
- Observe teacher demonstrations.
- Ask questions.
- Connect new knowledge with previous lessons.

## Teacher Activities

- Introduce lesson objectives.
- Explain key concepts.
- Demonstrate examples.
- Facilitate discussion.

## Student Activities

- Take notes.
- Participate in discussions.
- Observe demonstrations.
- Complete quick learning checks.

---

## Day 2 — Guided Practice

## Objectives — Guided Practice

Students apply newly learned concepts through structured activities.

### Activities

- Guided coding exercises
- Problem-solving worksheets
- Pair programming
- Algorithm design
- Small group discussions

### Teacher Role

- Provide guidance.
- Monitor progress.
- Clarify misconceptions.
- Encourage collaboration.

---

## Day 3 — Build

## Objectives — Build

Students build practical solutions using the week's learning.

### Example Activities

Programming

- Python application

Web Development

- Webpage design

Artificial Intelligence

- AI activity

Robotics

- Arduino circuit

Engineering

- Mini prototype

### Expected Outcome

Each learner produces a working implementation demonstrating the week's concepts.

---

## Day 4 — Improve

## Objectives — Improve

Students refine, test, and improve their work.

### Activities — Improve

- Debugging
- Code review
- Hardware testing
- Performance improvement
- Documentation updates

### Peer Learning

Students should:

- Review classmates' work.
- Offer constructive feedback.
- Identify improvement opportunities.
- Share alternative solutions.

---

## Day 5 — Present & Reflect

## Objectives — Present & Reflect

Students communicate what they learned and reflect on their progress.

### Activities — Present & Reflect

- Project presentation
- Live demonstration
- Question and answer session
- Reflection journal
- Weekly quiz

### Reflection Questions

Students should consider:

- What did I learn?
- What challenges did I face?
- How did I solve them?
- What can I improve next week?

---

## Weekly Deliverables

Every student should complete:

- Programming exercise
- Practical activity
- Documentation update
- Reflection journal
- Portfolio evidence

---

## Weekly Assessment

| Component | Description |
| ---------- | ------------- |
| Participation | Classroom engagement |
| Coding Exercise | Practical programming |
| Laboratory Activity | Hands-on implementation |
| Documentation | Markdown notes and reports |
| Reflection | Personal learning journal |

---

## Engineering Design Cycle

Every practical activity follows this engineering workflow.

1. Understand the Problem
2. Plan the Solution
3. Design
4. Build
5. Test
6. Debug
7. Improve
8. Present
9. Reflect

---

## Classroom Expectations

Students are expected to:

- Arrive prepared.
- Participate actively.
- Respect classmates.
- Follow laboratory safety rules.
- Write clean, readable code.
- Maintain project documentation.
- Ask thoughtful questions.
- Support collaborative learning.

---

## Teacher Responsibilities

Teachers should:

- Create an engaging learning environment.
- Demonstrate concepts clearly.
- Encourage experimentation.
- Provide timely feedback.
- Monitor individual progress.
- Promote teamwork.
- Foster creativity and innovation.

---

## Weekly Resources

Students may use:

- Python
- VS Code
- Arduino IDE
- HTML & CSS
- Classroom robotics kits
- Worksheets
- Presentation slides
- Documentation templates

---

## Expected Weekly Outcomes

At the end of each week, students should have:

- Learned at least one new technical concept.
- Completed one practical activity.
- Updated their learning portfolio.
- Reflected on their progress.
- Received constructive feedback.
