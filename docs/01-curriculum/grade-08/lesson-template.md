## Lesson Structure

## Phase 1 — Engage

### Duration

10 Minutes

### Teacher Activities

- Introduce the lesson.
- Connect with prior knowledge.
- Present a real-world problem.
- Explain lesson objectives.

### Student Activities

- Participate in discussion.
- Share ideas.
- Ask questions.

### Deliverables

- Class discussion
- Initial observations

---

## Phase 2 — Explore

### Duration  — Explore

20 Minutes

### Teacher Activities  — Explore

- Demonstrate concepts.
- Explain technical terminology.
- Show practical examples.

### Student Activities  — Explore

- Observe demonstrations.
- Take notes.
- Answer guided questions.

### Deliverables  — Explore

- Learning notes
- Practice exercises

---

## Phase 3 — Build

### Duration — Build

35 Minutes

### Teacher Activities — Build

- Guide practical work.
- Support students during implementation.
- Monitor progress.

### Student Activities — Build

- Write code.
- Build websites.
- Assemble robotics circuits.
- Test solutions.

### Deliverables — Build

- Working prototype
- Source code
- Circuit diagram (if applicable)

---

## Phase 4 — Improve

### Duration — Improve

15 Minutes

### Teacher Activities — Improve

- Review student work.
- Provide constructive feedback.
- Demonstrate debugging strategies.

### Student Activities — Improve

- Fix errors.
- Improve functionality.
- Refactor code.
- Optimize designs.

### Deliverables — Improve

- Improved solution
- Updated documentation

---

## Phase 5 — Present

### Duration — Present  

10 Minutes

### Teacher Activities — Present

- Facilitate presentations.
- Ask reflective questions.
- Encourage peer feedback.

### Student Activities — Present

- Demonstrate projects.
- Explain technical decisions.
- Reflect on learning.

### Deliverables — Present

- Presentation
- Reflection
