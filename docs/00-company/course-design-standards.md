## Course Design Standards

## Purpose

This document defines the official standards for designing educational offerings at Code and Core University.

Its purpose is to ensure every course provides a consistent, high-quality learning experience while remaining flexible enough to support subjects ranging from programming and artificial intelligence to robotics, data science, and corporate training.

These standards apply to all learning formats, including documentation, YouTube, self-paced courses, instructor-led bootcamps, workshops, and enterprise training.

---

## Relationship to Other Documents

This document builds upon the educational foundation established in previous company documents.

- **Mission Statement** defines why Code and Core exists.
- **Vision Statement** defines where the organization is heading.
- **Teaching Philosophy** defines how instructors teach.
- **Learning Methodology (CCLF)** defines how learners progress.

Course Design Standards define **how educational content is structured**.

---

## Design Principles

Every Code and Core course should:

- Follow a consistent architecture.
- Support practical learning.
- Encourage curiosity.
- Build progressively.
- Reinforce professional habits.
- Produce portfolio-ready outcomes.
- Support lifelong learning.

---

## Standardized Course Architecture

Every course follows the same high-level architecture regardless of its duration or subject.

The depth of each section may vary, but the structure remains consistent.

```text
Course
    ↓
Course Orientation
    ↓
Module 1
    ↓
Module 2
    ↓
...
    ↓
Capstone Project
    ↓
Course Completion
    ↓
Next Learning Path
```

---

## Flexible Size, Fixed Architecture

Courses are designed according to learning objectives rather than a fixed duration.

Examples:

|Course|Typical Scope|
|---|---|
|Scratch|Short introductory course|
|Python|Multi-module programming course|
|Robotics|Comprehensive practical course|
|AI Engineering|Professional program|

Regardless of size, every course follows the same architectural principles.

---

## Course Orientation

Every course begins with a concise orientation.

The orientation prepares learners before technical instruction begins.

A standard orientation includes:

- Welcome
- Course overview
- Learning outcomes
- Prerequisites
- Required software and tools
- Study recommendations
- Course roadmap
- Introduction to the Code and Core Learning Framework (CCLF)
- Community and support resources

---

## Module-Based Structure

Courses are divided into logical modules.

Each module focuses on developing a specific competency.

```text
Course
│
├── Orientation
├── Module 1
├── Module 2
├── Module 3
├── Capstone
└── Next Steps
```

Modules allow learners to make measurable progress while enabling instructors to organize content effectively.

---

## Standard Learning Module

Every module follows a consistent structure.

## Required Components

- Module Introduction
- Learning Objectives
- Lessons
- Practice Activities
- Module Summary
- What's Next

## Optional Components

Depending on the learning objectives, a module may also include:

- Mini Challenge
- Module Project
- Module Assessment
- Additional Resources

---

## Standard Lesson Template

Every lesson follows a consistent instructional flow.

```text
Lesson Title
    ↓
Learning Objective
    ↓
Why This Matters
    ↓
Concept Explanation
    ↓
Demonstration
    ↓
Hands-on Activity
    ↓
Key Takeaways
    ↓
Further Practice (Optional)
```

Each lesson should answer four questions:

- What am I learning?
- Why does it matter?
- How does it work?
- Can I apply it?

---

## Progressive Project System

Projects are integrated throughout the course rather than reserved only for the end.

Every project should introduce one new competency while reinforcing previously learned skills.

Example progression:

```text
Mini Project
    ↓
Module Project
    ↓
Integrated Project
    ↓
Capstone Project
```

Projects should increase in complexity as learners gain confidence and experience.

---

## Professional Portfolio Project Standard (PPPS)

Every major project should be suitable for inclusion in a learner's professional portfolio.

A standard project includes:

- Project Brief
- Requirements
- Planning
- Implementation
- Testing
- Documentation
- Version Control (Git)
- Presentation or Demonstration
- Reflection
- Future Improvements

The objective is to develop not only technical ability but also professional engineering practices.

---

## Continuous Competency Assessment

Assessment is continuous throughout the learning journey.

Competency is demonstrated through consistent application rather than a single examination.

Assessment evidence may include:

- Practice activities
- Mini challenges
- Module assessments
- Projects
- Capstone project
- Documentation quality
- Git usage
- Debugging process
- Code quality
- Communication
- Reflection

The emphasis is on mastery, growth, and professional readiness.
