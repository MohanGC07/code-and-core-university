## Academic Framework

## Purpose

The Academic Framework defines how learners progress from Grade 5 to Grade 12.

Rather than focusing only on examinations, the framework develops knowledge, practical skills, creativity, ethical responsibility, and professional readiness through continuous project-based learning.

---

## Vision

Every learner should graduate from Grade 12 with the confidence and practical ability to build software, AI applications, and technology solutions while maintaining strong ethical and professional standards.

---

## Mission

Provide a structured learning pathway that combines:

- Computing Fundamentals
- Programming
- Artificial Intelligence
- Robotics
- Software Engineering
- Career Readiness

through project-based education.

---

## Educational Philosophy

Learning follows the cycle:

Learn

↓

Practice

↓

Build

↓

Reflect

↓

Improve

↓

Share

↓

Lead

---

## Learning Principles

Students learn best when they:

- Build real projects
- Solve authentic problems
- Collaborate with others
- Learn by experimentation
- Receive continuous feedback
- Teach and mentor peers
- Reflect on their work

---

## Academic Journey

| Stage | Grades | Focus |
|--------|--------|----------------------------|
| Foundation | 5–6 | Digital literacy, logical thinking, creativity |
| Explorer | 7–8 | Programming, robotics, web fundamentals |
| Builder | 9–10 | Software engineering, AI foundations, teamwork |
| Professional | 11–12 | AI, full-stack development, research, portfolio |

---

## Graduate Outcomes

By completing Grade 12, students should be able to:

- Write clean, maintainable code.
- Build complete software applications.
- Understand AI fundamentals and apply AI tools responsibly.
- Design and build robotics projects.
- Work effectively in teams.
- Use Git and collaborative development workflows.
- Communicate technical ideas clearly.
- Create and maintain a professional portfolio.
- Continue into university-level computing with confidence.

---

## Learning Areas

Every learner studies across multiple schools throughout the program.

- School of Programming
- School of Computing
- School of Software Engineering
- School of Artificial Intelligence
- School of Robotics
- School of Career Development

---

## Annual Learning Model

Each academic year includes:

- Concept Learning
- Guided Practice
- Mini Projects
- Major Projects
- Assessments
- Portfolio Updates
- Reflection Activities

---

## Assessment Philosophy

Assessment measures understanding and application.

Assessment methods include:

- Practical assignments
- Coding exercises
- Robotics demonstrations
- Quizzes
- Team projects
- Presentations
- Portfolio reviews
- Capstone projects

---

## Portfolio Requirement

Every learner maintains a portfolio containing:

- Programming projects
- Robotics projects
- AI projects
- Documentation
- Presentations
- Git repositories
- Reflection journals

The portfolio grows from Grade 5 through Grade 12.

---

## Technology Progression

Technology is introduced gradually.

Grades 5–6

- Scratch
- PictoBlox
- Basic Robotics

Grades 7–8

- Python
- HTML
- CSS
- Arduino

Grades 9–10

- JavaScript
- SQL
- Git
- Raspberry Pi
- APIs

Grades 11–12

- React
- Next.js
- FastAPI
- PostgreSQL
- Docker
- Machine Learning
- Deep Learning
- Computer Vision
- Agentic AI

---

## Graduation Requirements

To complete the program, learners should:

- Complete the curriculum.
- Pass required assessments.
- Maintain a project portfolio.
- Complete a capstone project.
- Demonstrate teamwork and professional conduct.
- Present their final work.

---

## Continuous Improvement

The academic framework is reviewed annually based on:

- Student outcomes
- Industry developments
- Instructor feedback
- Parent feedback
- Technological changes
